# Shared Feature: Language / i18n

Required structure and behavior:
- The interface supports `English (en-US)`, `繁體中文 (zh-TW)`, and `日本語 (ja-JP)`.
- On first use, the interface follows the browser language when it is supported and
  otherwise uses English.
- The selected language is saved and used on later visits.
- The [Home](../components/home.md) view displays a language selector using each
  language's native name.
- Selecting a language immediately redraws the interface in that language.
- The top bar, sidebars, menus, page titles, controls, forms, dialogs, status text,
  notifications, and GUI-generated messages use the selected language.
- Profile names, server names, paths, console commands, user-entered text, and server
  process output remain unchanged.
- Missing translations fall back to English instead of displaying an empty label or
  breaking the page.
- Translated labels must fit their controls without clipping, overlapping, or changing
  the navigation layout.
- Every UI-visible label needs a matching key in every catalog under
  `module/webui/locales/`, with identical `{placeholder}` names in each.

**Tests:** `tests/test_i18n.py` (catalog fallback/placeholder parity, persistence),
`tests/test_gui_playwright.py` (detection/switch/persistence round-trip).
